PROBLEM SOLVING medium
Describe the typical steps involved in the 'Logical Design' phase of database design, following the conceptual design.

Λύση:

  1. Step 1: The ER (Entity-Relationship) model, created during conceptual design, is converted into the data model of the specific database system (e.g., relational, object-oriented, hierarchical).
  2. Step 2: This conversion results in the logical schema of the database.
  3. Step 3: Normalization is typically applied to optimize the logical schema, reducing data redundancy and improving data integrity.

PROBLEM SOLVING medium
You are tasked with improving the performance of an existing DBMS application. Identify two distinct levels at which you can attempt to 'tune' the DBMS, and provide one specific example for each level.

Λύση:

  1. Level 1: Hardware Level. Example: Adding faster disks (e.g., SSDs) to improve I/O speed, or increasing the amount of main memory to allow for better buffer utilization.
  2. Level 2: DBMS Parameters Level. Example: Adjusting the size of the buffer pool (e.g., buffer_pool_size in MySQL) to minimize page faults, or configuring checkpointing frequency to limit the size of the transaction log file.
  3. Level 3: High-level Design Choices Level. Example: Optimizing the logical schema through normalization, or creating/modifying indexes (e.g., B+ trees) on frequently queried columns to speed up data access.

PROBLEM SOLVING hard
A small company initially managed its customer data using simple text files. As the company grew, they started experiencing several issues. Based on the lecture, identify three specific problems they likely encountered due to using a file system instead of a DBMS, and briefly explain why each is a problem.

Λύση:

  1. Problem 1: Redundancy and Inconsistency. Explanation: The same customer information might be repeated in different files, leading to increased storage cost, higher retrieval cost, and difficulty in updating consistently.
  2. Problem 2: Difficulty of Access. Explanation: File systems lack efficient mechanisms for searching and retrieving information in various ways, and data scattered across files with different formats makes complex queries difficult without custom programming.
  3. Problem 3: Atomicity Problems. Explanation: If a multi-step operation fails midway, the system might be left in an inconsistent state without a built-in mechanism to revert to a consistent state.
  4. Problem 4 (Alternative): Concurrent Access Anomalies. Explanation: When multiple users try to read and update the same data simultaneously, inconsistencies can arise as file systems lack mechanisms to control this.
  5. Problem 5 (Alternative): Security Problems. Explanation: File systems typically offer limited security controls (e.g., file-level permissions), making it difficult to grant different users different access rights to specific data within a file.
  6. Problem 6 (Alternative): Integrity Problems. Explanation: Data consistency constraints are enforced by application programs, not the file system, requiring modification and recompilation of all relevant application programs for changes.

  • identifies_three_distinct_problems: 45 βαθμοί
  • provides_clear_and_correct_explanation_for_each_problem: 55 βαθμοί
Θέματα: File system limitations, DBMS advantages
SHORT ANSWER QUESTION easy
Briefly explain the difference between 'Physical Data Independence' and 'Logical Data Independence' in a DBMS.

Σημαντικά σημεία:

  • Physical Data Independence: Allows changes to the physical storage structures (e.g., adding/removing indexes, changing file organization) without affecting the logical organization of the data or the application programs.
  • Logical Data Independence: Allows modifications to the logical structure of the data (e.g., adding new relations/tables, adding new fields/columns to existing tables) without affecting the application programs at the external level (user views).
Μέγιστη βαθμολογία: 10
Θέματα: Data independence
SHORT ANSWER QUESTION medium
Describe the role of the 'Disk Space Manager' and how it contributes to the overall efficiency of a DBMS.

Σημαντικά σημεία:

  • Role: Responsible for allocating and deallocating disk space, and for reading/writing data pages (the unit of data) to and from the disk.
  • Contribution to Efficiency: It ensures the efficient use of available disk space and provides fast access to the database files. By managing pages, it abstracts away the complexities of physical disk storage from higher-level DBMS components, allowing them to assume efficient disk operations.
Μέγιστη βαθμολογία: 10
Θέματα: DBMS architecture, Disk Space Manager
SCENARIO BASED QUESTION medium
A new startup is building a social media platform. They anticipate a massive volume of user-generated content, including unstructured data like photos and videos, and expect very high rates of data ingestion and retrieval. They are concerned about scalability and speed, especially given the rapid growth seen in similar platforms (the 'data avalanche' phenomenon).
Based on the historical evolution of databases discussed in the lecture, would you recommend a traditional Relational Database Management System (RDBMS) or a NoSQL database for their primary data store? Justify your choice by referencing relevant concepts from the lecture.

Βασικά σημεία ανάλυσης:

  • User-generated content
  • unstructured data
  • massive volume
  • scalability
  • speed
  • data avalanche
  • RDBMS limitations
  • NoSQL advantages

Ενδεικτική απάντηση:

For a social media platform with massive, unstructured, user-generated content and high demands for scalability and speed, a NoSQL database would be the more appropriate choice for their primary data store. The lecture highlights that traditional relational databases 'struggled with scalability and speed required' for the demands of the 'Web 2.0 revolution' and the 'data avalanche' phenomenon, which brought about a shift to user-generated content and largely unstructured data. NoSQL databases, which existed since the 1960s but gained prominence with Web 2.0, were developed to address these specific challenges. They use different data structures (like key-value, graph, document) that are often better suited for handling large volumes of unstructured or semi-structured data and providing horizontal scalability for high throughput. While RDBMS still have their place for reliability and consistency in business systems, the core requirements of this scenario align more closely with the strengths of NoSQL.
Θέματα: Database history, NoSQL, RDBMS, Big Data, scalability, unstructured data
CALCULATION QUESTION medium
A DBMS is experiencing a bottleneck due to high CPU utilization. If the 'Golden Rule' for system design suggests a maximum utilization of 70% for optimal performance, and the current CPU utilization is 95%, what is the percentage reduction in CPU load required to bring it down to the recommended maximum?
Δεδομένα: current_cpu_utilization: 95%, recommended_max_cpu_utilization: 70%

Βήματα επίλυσης:

  1. 1. Calculate the absolute difference in utilization percentage points: 95% - 70% = 25%.
  2. 2. Calculate the percentage reduction relative to the current load: (Absolute difference / Current utilization) * 100.
  3. 3. (25 / 95) * 100 ≈ 26.3157...
Τελική απάντηση: Approximately 26.32% reduction in current CPU load.
